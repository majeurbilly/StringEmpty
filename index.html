Membre 2

Billy Hallé

Rôle : Développeur Backend & Lead Tech

Focus principal de cette itération :

Mon objectif exclusif pour ce sprint était le développement de l'API C# pour tout le module Utilisateurs/RH.
J'ai implémenté la logique d'affaires, appliqué une politique de sécurité stricte (RBAC) et préparé la totalité des points d'accès de données dont le Frontend avait besoin pour construire les interfaces.

🎬 L'API en Action (Swagger)

🛠️ Réalisations Techniques (Backend)

Opérations CRUD Utilisateurs

- API pour Ajouter, Modifier, Supprimer et Lister les utilisateurs.
- Préparation des données spécifiques pour la page RH et le Dashboard.

Sécurité & Rôles (RBAC)

- Implémentation stricte des rôles : restriction des actions critiques aux Admins/Gérants.
- Validation des droits (401/403) et endpoints pour modifier/consulter son propre profil.

Fondations pour le Frontend

- Création des services nécessaires pour alimenter le Layout principal et le Header (informations de l'utilisateur connecté).

Documentation & Fiabilité

- Exposition de Swagger pour faciliter l'intégration frontend.
- Couverture massive de tests d'intégration (plus de 50 tests) validant chaque règle d'affaires.

🎯 Résultats & Prochaines étapes

Résultat Final (Itération 2)

Le backend du module RH/Utilisateurs est 100% opérationnel, documenté et testé rigoureusement. La plomberie est prête pour brancher l'interface utilisateur.

À attaquer pour l'Itération 3

  • Développer les points de terminaison (API) pour le module complet de l'Inventaire.
  • Assister l'équipe sur la connexion du Frontend React avec nos routes sécurisées.
