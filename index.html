Membre 1

Antony Burlet

Rôle : Développeur Frontend

Focus principal de cette itération :

Cette itération, mon travail s'est concentré sur trois gros blocs frontend : le CRUD des succursales, le CRUD des items (produits et livres), et le ménage de la sidebar pour rendre la navigation plus claire et plus maintenable.

📋 Gros livrables

  • 1
    CRUD des succursales côté frontend

    Créer, consulter, modifier et supprimer des succursales depuis l'interface.

  • 2
    CRUD des items

    Gestion des produits et des livres, avec les écrans nécessaires pour manipuler le catalogue.

  • 3
    Ménage de la sidebar

    Réorganisation de la navigation pour mieux refléter les modules et réduire le bruit visuel.

🧩 Travail frontend réalisé

🚀 Défis

Défis rencontrés

  • • Distinguer clairement produits et livres dans le CRUD des items
  • • Garder les formulaires simples malgré plusieurs types de données
  • • Nettoyer la sidebar sans briser la navigation existante
  • • Brancher les écrans au reste de l'application de façon cohérente

Solutions appliquées

  • • Séparer les responsabilités entre les écrans et les formulaires
  • • Réutiliser les patterns frontend déjà présents dans le projet
  • • Regrouper la navigation par module dans la sidebar
  • • Tester les parcours principaux directement dans l'interface
Membre 2

Billy Hallé

Rôle : Développeur Full-Stack

Focus principal de cette itération :

Mon objectif a été de concevoir et livrer la totalité du module Ressources Humaines en mode de bout en bout (End-to-End).

Du modèle de données Entity Framework jusqu'aux formulaires React interactifs, j'ai implémenté la logique métier complexe (chevauchements de dates, paie) tout en sécurisant l'application avec plus de 230 tests automatisés au total et un ajustement rigoureux des permissions d'accès.

🎬 Le Module RH & Inventaire en Action

🛠️ Réalisations Techniques (Full-Stack)

Machine à États & Logique Métier

- Flux d'approbation strict pour les feuilles de temps et les absences.
- Validation algorithmique des chevauchements temporels (contrats, congés) avec messages 409 localisés.

Interfaces React Dynamiques

- Création de 8 pages métier (Annuaire, Paie, Dashboard KPI, etc.).
- Utilisation de useQuery pour des formulaires à dépendances croisées (ex: Assigner un rôle à un employé).

Assurance Qualité Totale

- Cap des 200 tests d'intégration backend dépassé.
- Mise en place de Vitest et React Testing Library sur le frontend pour garantir la stabilité des formulaires.

Refonte RBAC Frontend/Backend

- Correction du mur "403 Forbidden" sur l'inventaire.
- Alignement des droits côté React via des requêtes allégées (getDropdown) respectant le principe du moindre privilège.
